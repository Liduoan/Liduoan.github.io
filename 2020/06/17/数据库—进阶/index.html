范式

数据库设计的范式：设计数据库时需要遵循的一些规范

设计关系型数据库的时候，遵从不同的规范要求，设计出合理的数据库

目前数据库有六种范式——主要介绍前三种

设计数据库时，要遵循后面的范式，要求先遵循前面的范式

学习了这些范式招式，是为了在后面设计数据库更加合理

第一范式1NF

每一列都是不可分割的原子数据项

也就是说每一列不可以分割成更小的

最基本的要求

第一范式也有问题存在：

数据冗余特别严重

数据添加存在问题——某些列不存在某些列存在

数据删除也存在问题——某些列删除了 会一起把另外一些列也删除了

第二范式2NF

在第一范式的基础上，非码属性必须完全依赖于码

在1NF基础上消除非主属性对主码的部分函数依赖

  • 函数依赖：A—>B，如果通过A的属性值或者属性组的值，可以确定唯一B属性的值 则称B依赖于A

    学号—->姓名 （学号，课程名称） —->分数

  • 完全函数依赖：A——->B 如果A是一个属性组，则B属性值的确定需要依赖A属性组中所有的属性值

    （学号，课程名称）—->分数

  • 部分函数依赖： A——> B 如果A是一个属性组，则B属性值的确定只需要依赖于A属性组中某一些值即可

    （学号，课程名称）——> 姓名

  • 传递函数依赖：A—->B, B—->C. 如果通过A属性的值可以确定唯一的B属性值。在通过B属性的值可以确定唯一C属性的值，则称C传递函数依赖于A

    学号—->系名——>系主任

  • 码：如果在一张表中，一个属性或属性组，被其他所有属性完全依赖，则称这个属性为该表的码

    例如，该表中的码为（学号，课程名称)

    主属性：码属性组中所有的属性

    非主属性：除了码属性组的属性

所以找到码 然后满足范式要求

第三范式3NF

在2NF的基础上，任何非主属性不依赖于其他非主属性

在2NF的基础上消除传递依赖

事务

事务的基本介绍

概念：一个包含多个步骤的业务操作，被事务管理，那么这些操作要么同时成功，要么同时失败

操作：

所以综合来说三个操作

1、开启事务 start transaction;

2、回滚操作 rollback;

3、提交 commit;

Mysql数据库中事务默认自动提交

一条DML（增删改）语句自动提交一次事务，数据会被持久化的更新。

事务提交的两种方式：

自动提交：

​ mysql就是自动提交的

​ 一条DML（增删改）语句自动提交一次事务

手动提交：

​ Oracle数据库是手动提交的，也就是说增删改语句之后一定要提交才会生效

​ 需要开启事务，再提交

修改事务的默认提交方式：

查询事务的默认提交方式：select @@autocommit; 1表示自动提交 0表示手动提交

修改默认提交方式： set @@autocommit = 0;

事务的四大特征

1、原子性：事务是原子的 要么同时成功 要么同时失败 是不可分割的最小单位i

2、持久性：如果事务一旦提交或者回滚，会持久更新数据库表。也就是说数据库被改变了。

3、隔离性：多个事务之间，相互独立

4、一致性：事务操作前后数据总量不变 ——比如张三和李四金钱的总数不变

事务的隔离级别

概念：多个事务之间，相互独立，隔离的。但是如果多个事务操作同一批数据，则会引发一些问题，设置不同的隔离级别就可以解决这些问题。

存在的问题：

1、脏读

一个事务，读取到另一个事务中没有提交的数据

例如： A事务有转账操作 未提交 B事务查看数据 发现转账成功了

问题：但是A事务没有提交 如果A事务遇到回滚，那么数据库中转账就失败了。

2、不可重复读（虚读）

在同一个事务中两次读取到的数据不一样

例如 ： A 事务有转账操作 未提交 B事务查看数据 没有更新数据 。然后A事务提交，B事务查看数据，数据更新了。

问题：某些事务要求B事务查询的是一样的。

3、幻读

一个事务操作（DML）数据表中的所有记录，另一个事务添加一条数据，则第一个一个事务查询不到自己的修改

隔离级别：
1、read uncommitted:读未提交

产生的问题：脏读，不可重复读，幻读

2、read committed ：读已提交（Oracle默认）

产生的问题：不可重复读，幻读

3、repeatable read: 可重复读 （Mysql默认）

产生的问题：幻读

4、serializable：串行化

串行化：锁表的动作，一个事务在操作一个数据表，另外一个事务 不可以操作这个数据表

可以解决所以的问题

注意：

隔离级别从小到大安全性越来越高，但是效率越来越低

数据库查询隔离级别：

select @@tx_isolation;

数据库设置隔离级别

set global transaction isolation level 级别字符串;
